# ADB Enterprise Utility Pro

**版本**: v2.0.0  
**日期**: 2026-05-22  
**作者**: Magic  
**适用环境**: Python 3.8+ | Windows / Linux / macOS

---

## 一、工具概述

ADB Enterprise Utility Pro 是一款面向 Android 运维与测试场景的企业级终端工具。基于纯 Python 标准库开发，零第三方依赖，提供设备管理、应用生命周期管理、文件传输、屏幕媒体、输入模拟、系统信息监控、日志调试、端口转发等 12 大功能模块，支持 USB 与网络 ADB 双模式。

---

## 二、环境要求

| 项目 | 要求 | 备注 |
|------|------|------|
| Python | 3.8 及以上 | 仅需标准库 |
| ADB | Android SDK Platform-Tools | 必须加入系统 PATH |
| 操作系统 | Windows 10+/Linux/macOS | 终端需支持 ANSI 色彩 |

**验证命令**：
```bash
python --version   # ≥ 3.8
adb version        # 能正常输出版本号
```
若未安装Platform-Tools，请参考requirements.txt

## 三、快速开始

### 3.1 解压与启动

```bash
# 进入工具目录
cd adb_tool_pro

# 直接运行
python adb_tool_pro.py
```

### 3.2 首次使用流程

1. **连接设备**：通过 USB 连接手机并开启「开发者选项 → USB 调试」；或通过 WiFi 使用 `adb connect IP:5555`。  
2. **选择设备**：启动工具后按 `9` 进入「切换/选择设备」，输入序号或 IP:Port 完成绑定。  
3. **执行操作**：返回主菜单，按对应数字键进入功能子菜单。  
4. **退出程序**：主菜单按 `0` 安全退出。

## 五、详细使用指南

### 5.1 设备管理

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| 1 | 列出所有设备 | 显示序列号、状态、型号、Transport ID |
| 2 | 连接网络设备 | 输入 IP 与端口（默认 5555）建立无线 ADB |
| 3 | 断开网络设备 | 支持指定断开或全部断开 |
| 4 | 设备详细信息 | 读取 build.prop 关键属性 |
| 5 | 重启设备 | 需用户确认 |
| 6 | 重启到 Recovery | 需用户确认 |
| 7 | 重启到 Bootloader | 需用户确认 |
| 8 | 导出 Bugreport | 采集完整系统诊断报告，保存至 `~/.adb_tool_bugreports/` |

### 5.2 应用管理

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| 1 | 列出所有应用 | 仅展示前 50 条，含系统/第三方标识 |
| 2 | 列出系统应用 | 过滤 `pm list packages -s` |
| 3 | 列出第三方应用 | 过滤 `pm list packages -3` |
| 4 | 应用详情透视 | 展示权限、Activity、签名、版本信息 |
| 5 | 安装 APK | 支持 `-r` 覆盖安装 |
| 6 | 批量安装 APK | 遍历目录下所有 `.apk` 文件，带进度条 |
| 7 | 卸载应用 | 标准卸载 |
| 8 | 清除应用数据 | 等效 `pm clear` |
| 9 | 启动应用 | 通过 Monkey 或指定 Activity 启动 |
| 10 | 冻结应用 | `pm disable-user --user 0` |
| 11 | 解冻应用 | `pm enable` |
| 12 | 提取 APK | 导出已安装应用原始 APK 至 `~/.adb_tool_apks/` |

### 5.3 屏幕与媒体

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| 1 | 截图 | 保存至 `~/.adb_tool_screenshots/` |
| 2 | 截图并打开 | 截图后自动调用系统默认查看器 |
| 3 | 录屏 | 默认 10 秒，可自定义比特率 |
| 4 | 输入模拟 | 子菜单支持点击/滑动/文本/按键/分辨率获取 |

**输入模拟常用按键对照**：
- `HOME` / `BACK` / `POWER` / `MENU` / `ENTER`
- `VOLUME_UP` / `VOLUME_DOWN` / `SPACE` / `DEL` / `RECENT`

### 5.4 系统信息 — 实时性能监控

进入 `5 → 7`，工具将以指定间隔轮询设备，实时渲染终端仪表盘：

- **CPU**：使用率条形图 + Sparkline 趋势折线、核心数/频率/温度
- **内存**：总/已用/可用、ZRAM/Swap 占用
- **电池**：电量、温度、电压、电流、状态、健康度
- **存储**：内部/外部存储使用率表格
- **网络**：WiFi SSID、信号强度分级、IP 地址
- **GPU**：频率（Qualcomm/Mali 双适配）
- **前台应用** 与 **Top 5 进程**

按 `Ctrl+C` 提前结束监控。

### 5.5 日志与调试 — 端口转发

| 快捷键 | 功能 | 命令映射 |
|--------|------|----------|
| 1 | 列出转发规则 | `adb forward --list` / `reverse --list` |
| 2 | 添加端口转发 | `adb forward tcp:local tcp:remote` |
| 3 | 添加反向转发 | `adb reverse tcp:remote tcp:local` |
| 4 | 清除转发规则 | 支持单条清除或 `--remove-all` |

### 5.6 文件浏览器

进入 `3 → 3`，进入设备端交互式 Shell 文件浏览器：

```
命令集：
  ls              列出当前目录
  cd <path>       切换目录（支持相对/绝对路径）
  pwd             显示当前路径
  cat <file>      查看文件内容（自动截断 3000 字符防刷屏）
  pull <file> [local]  拉取文件到本地
  exit            退出浏览器
```

## 七、注意事项

1. **权限**：部分操作（如冻结系统应用、查看其他应用详细信息）可能需要 Root 权限或 Android 13+ 特殊权限，失败时请以错误提示为准。  
2. **网络 ADB**：无线调试需设备与 PC 处于同一局域网，且提前通过 USB 执行 `adb tcpip 5555` 开启端口。  
3. **Windows 终端**：Windows 10 以下版本可能无法正确显示 ANSI 炫彩字符，工具已自动尝试启用虚拟终端处理，若仍异常请使用 Windows Terminal 或 PowerShell 7+。  
4. **性能监控**：`dumpsys cpuinfo` 与 `meminfo` 输出格式因厂商定制存在差异，解析失败时会自动降级到 `/proc` 文件系统读取。  
5. **单文件设计**：所有功能封装于单一 `.py` 文件，便于直接复制到任何已配置 Python 的环境运行，无需 pip 安装。
